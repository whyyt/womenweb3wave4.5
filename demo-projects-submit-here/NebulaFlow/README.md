# **让努力和坚持获得回报，让你的旅程能被记录**

## ✨ 核心理念

一个把「参加活动 / 坚持打卡」变成 **可验证链上记录** 的 Web3 平台。用户可以加入长期挑战或集会活动，用钱包参与、签到和结算 ，有的活动用 **押金奖池** 激励长期坚持，有的活动用 **NFT 勋章** 记录参与痕迹，所有这些会汇总到你的 **个人链上成长档案（My Journey）** 里。

NebulaFlow 不只是一个打卡 App，而是你的 **Web3 行动记录簿**。在这里：

- 长期挑战  → 使用 **押金奖池模式（Stake Pool）**：参与者缴纳押金，坚持完成挑战的人平分奖池。

- 集会性活动（某天的 meetup、线下聚会等）  → 使用 **NFT 模式**：活动结束后为合格参与者发放 **专属 NFT 勋章**。

# 💡 解决的痛点

1. **动力不足与低完成率**
   - **传统模式：** 单纯依靠意志力，缺乏持续的外部激励，极易中途放弃。
   - **NebulaFlow：** 引入真实的经济激励（押金与奖金），将“我要坚持”转变为“坚持就能赚钱”，极大提升参与感和完成欲。
2. **信任与透明度缺失**
   - **传统模式：** 中心化平台管理资金和规则，存在资金挪用、规则暗改的风险。
   - **NebulaFlow：** 所有资金和核心规则由开源智能合约托管，链上公开可查，实现真正的去中介化和程序正义。
3. **成就记录无法验证**
   - **传统模式：** 完成记录存储在中心化服务器，缺乏公信力和永久性。
   - **NebulaFlow：** 成功完成的挑战将以 **NFT** 的形式永久记录在您的链上地址中，构成可验证的、属于自己的成就档案。

## ⚙️核心功能

### 对于挑战参与者

- **🦊 Web3 钱包登录:** 使用 MetaMask 等钱包无缝连接，无需繁琐注册。
- **🔍 探索挑战:** 浏览各类公开的押金挑战活动。
- **💰 支付押金参与:** 一键支付押金，等待挑战开启。
- **✅ 轻松签到/验证:** 在挑战期间，通过简单的每日签名或提交证明来完成打卡。
- **🏆 瓜分奖池 & 获取 NFT:** 成功完成挑战，自动从智能合约中领取奖金和独一无二的纪念性 NFT 成就徽章。

### 对于挑战发起者

- **🚀 创建挑战:** 轻松设置挑战名称、描述、周期、押金金额和打卡规则。
- **👥 管理活动:** 查看参与者状态，在特殊情况下可提前终止活动。

### 平台特性

- **🤖 智能合约自动结算:** 淘汰、奖金分配完全自动化，无需人工干预。
- **🔒 资金安全:** 用户押金直接锁定在经过审计的智能合约中，平台无法触碰。
- **📊 透明的链上记录:** 所有交易和状态变更均在区块链上可追溯。

## 🏗️ 技术架构

### 前端（UI）

- **框架**：Next.js
- **UI & 样式**：React + Tailwind CSS
- **动画**：GSAP（用于过渡动画和交互动效）
- **Web3 交互**：Ethers.js / wagmi
- **钱包**：MetaMask 等 EVM 兼容钱包

主要页面包括：

- `Activity Hub` – 活动总览与创建入口
- `Activity Detail` – 活动详情 / 报名 / 签到 / 结束等操作
- `My Journey` – 个人档案三分类展示

---

### 主要智能合约（Contracts）

- **ChallengeFactory.sol**
  - 负责创建和管理所有活动
  - 支持两种挑战类型：
    - `createDepositChallenge(...)` – 押金奖池挑战
    - `createNFTChallenge(...)` – NFT 活动
  - 核心职责：
    - 创建挑战
    - 用户报名 / 押金收取
    - 每日签到逻辑
    - 结束挑战后结算奖池或触发 NFT 铸造

- **RewardNFT.sol**
  - 遵循 ERC-721 标准
  - 为完成活动的用户铸造 NFT 勋章
  - 可配置基础 Token URI / 元数据

## 🔗 用户视角下的一次完整流程

1. **创建活动**
   - 主办方在 Activity Hub 页填写活动表单
   - 选择「押金模式」或「NFT 模式」
   - 前端调用 `ChallengeFactory` 合约创建活动

2. **用户报名**
   - 用户连接钱包
   - 点击「报名 / 参与」
   - 押金模式：通过 `msg.value` 转入押金
   - 合约记录该用户为参与者

3. **活动签到**
   - 活动期间：
     - 用户进入活动详情页
     - 点击「签到」
     - 合约更新该用户签到进度与状态

4. **活动结束**
   - 主办方点击「结束活动」：
     - 押金模式：筛选出完成挑战的用户，按人头均分奖池
     - NFT 模式：调用 `RewardNFT` 为合格参与者铸造 NFT

5. **My Journey 展示**
   - 前端从链上读取与当前钱包相关的活动数据
   - 按规则分到 **Professional Web3 / Social Web3 / Lifestyle** 三类
   - 展示进行中的预览卡片和已完成的 🏆 完成卡片
